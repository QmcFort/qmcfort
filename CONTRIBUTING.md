# How to Contribute

Contributions to **QmcFort** are very welcome!

Whether you want to report a bug, request a feature, improve documentation, or contribute code, 
please follow the steps below to ensure a smooth process.

## 1. Reporting Issues

If you encounter a problem or have a question:
1. Open an issue on the GitHub repository
2. Include:
    - a clear description of the issue,
    - steps to reproduce it,
    - your compiler/toolchain,
    - relevant input files or error messages.

This helps us diagnose and fix problems quickly.

## 2. Requesting New Features

If you have an idea for a new feature or extension:
1. Open an issue labeled `feature request`.
2. Describe the use case and the scientific/technical motivation.
3. We can then discuss the design before implementation.

## 3. Contributing Code

To contribute patches or new functionality:
1. Fork the repository and create a feature branch
2. Make your changes following the code's style
3. Ensure QmcFort compiles with new changes
4. Add or update unit/regression tests when relevant
5. Ensure all tests pass
6. Submit a pull request on GitHub
